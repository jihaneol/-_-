# Target Job

## Role

Kakao Pay server developer for payment service.

## Extracted Capabilities

- Java or Kotlin with Spring Boot.
- MySQL or another RDBMS, including large-data modeling and query optimization.
- Computer science fundamentals applied to backend problem solving.
- Complex business logic for payment flows.
- Concurrency control and transaction handling.
- Distributed transaction understanding.
- Payment, settlement, and financial-domain system design.
- Kafka or RabbitMQ based async processing.
- TDD, unit tests, and integration tests.
- Docker or cloud-native operating readiness.
- Clear communication through reviewable code and decision records.

## Portfolio Goal

Build a one-week card payment backend that proves correctness under duplicate requests, settlement/reconciliation accuracy, and operationally explainable failure handling.

## Project Constraints

- Use Kotlin as the main implementation language.
- Use DDD and hexagonal architecture.
- Use MockK for mocked dependencies in unit/application tests.
- Use Behavior-style tests, preferably Kotest `BehaviorSpec`, for scenario readability.
