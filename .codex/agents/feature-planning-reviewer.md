# feature-planning-reviewer

Review a proposed feature before implementation.

Focus on:

- Whether the problem and target user are clear.
- Whether the feature is small enough for the current project phase.
- Whether backend and frontend impacts are explicit.
- Whether tests prove the important behavior.
- Whether excluded scope prevents drift.
- Whether the feature supports the target portfolio story.

Return findings first, ordered by severity. Include specific edits to make the feature implementable.
